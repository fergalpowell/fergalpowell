Safer Walking

Oct 2018 - Feb 2019

Safer Walking is a cross-platform hybrid app that I designed and implemented as the sole developer for Ravencourt Ltd. The aim of the app is to help keep senior citizens safe using GPS tracking, Geofencing, SOS notifications and two-way voice communication. The front-end of Safer Walking app was built using the Cordova Phonegap framework which interacts with the backend GPS tracking server hosted on a Digital Ocean VPS via a REST API.

Technologies used: Cordova Phonegap, Traccar, MySQL, Digital Ocean, Android Studio, Xcode.

Safer Walking Registration

March 2019

I Designed, developed and deployed a registration web app for the Safer Walking mobile app. The web app consists of three pages: Home, Register and Account page. There are a number of REST API's integrated in the back-end of the web app.

  • Braintree API: A PayPal subsidiary, which is used to load a secure form to take the customer's card details and set up a recurring subscription.
  • Sendgrid API: This is used to send a confirmation email to the customer once they have successfully subscribed.
  • Twilio API: A cloud communications PaaS used to remotely configure customer's GPS trackers through SMS messages.
  • Traccar API: This is used to create and manage the customer's Safer Walking account.

Technologies used: Django, PostgreSQL, Digital Ocean, Braintree API, Twilio API, SendGrid API, Traccar API

Visit: https://www.saferwalkingactivation.com/

Dude Where's My Bike

Sep 2017 - May 2018

GPS bicycle tracking web application that assists the user in the recovery and prevention of bicycle theft. The system uses machine learning processes to detect frequently visited known locations of the user based on sensor data collected from a GPS tracking device. The system sets and monitors geofences on each known location and alerts the user if the bicycle leaves a geofence.

Technologies used: Django, Python, PostreSQL, PostGIS, Linux, Javascript, AWS, Docker, Vultr, Gammu, Overpass API.

On Yer Bike

Sep 2017 - Dec 2017

For the module, Advanced Web Mapping, I was required to create a Cordova mobile application that used a Django backend service to store data and incorporated some sort of location/mapping aspect. I chose to create a route planning tool for cyclists. The mobile app would allow the user to input a start and finish location in the Dublin city centre on a map. The app would then produce and display the quickest route from start to finish and also display the closest public bicycle locking station to the route destination on the map.

Technologies used: Python, Django, Javascript, LeafletJS, HTML, CSS, Docker, AWS, Git, Digital Ocean.

App Developer

Ravencourt Ltd

Stamford, United Kingdom

Sep 2018 - Feb 2019 • 6 months

As App Developer, I was responsible for designing, implementing and deploying the entirety of the app. By communicating well with the client to fully understand what they wanted, I produced clear and achievable requirements for the app.

Once the app requirements were detailed, I began researching and evaluating the suite of technologies that would be used to achieve these requirements. When evaluating technologies I looked at a number of different factors including: cost, implementation process, support, accessibility as well as my own personal familiarity with the technology.

I followed an Agile development process during implementation which allowed for requirements to be added/removed throughout the development cycle which meant the app was constantly evolving. Requirements were implemented based on their overall importance to the app as well as the level of risk associated with it. Each requirement was implemented, integrated and tested before development of the next requirement began.

After a number of iterations, the app was ready for deployment which involved ensuring that the app was properly configured and adhered to Apple and Google's guidelines. Once approved and made available in the app store, I monitored user feedback and released a number of small updates to fix any bugs encountered by users.

Before finishing up with Ravencourt, I wrote up clear and simple documentation for the client. Within this documentation, I included troubleshooting, administrative and maintenance instructions for the app and server.
